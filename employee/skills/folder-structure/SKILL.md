# Folder Structure Skill

## Employee Script Library Layout

```
employee/
├── AGENTS.md                         # Agent identity, workflow, and output format
├── employee.py                       # Main agent entry point (run this file)
├── tools.py                          # Search tools (grep, index, similarity, reader)
├── build_index.py                    # Index builder — scans scripts/ and creates scripts_index.json
│
├── scripts/                          # Old Selenium regression test scripts (~400 .py files)
│   ├── test_login.py
│   ├── test_record_creation.py
│   ├── test_approval_workflow.py
│   └── ...                           # All .py Selenium test files go here
│
├── scripts_index.json                # Pre-built index (auto-generated by build_index.py)
│                                     #   Contains: filename, class names, method names,
│                                     #   docstrings, imports, XPath count, line count
│
├── runs/                             # Saved agent output (written during runs)
│   └── <timestamp>_<slug>.txt
│
└── skills/                           # Progressive skill disclosure
    ├── folder-structure/
    │   └── SKILL.md                  # This file — explains the layout
    ├── index-lookup/
    │   └── SKILL.md                  # How to use the script index for fast lookup
    └── script-finder/
        └── SKILL.md                  # Main skill — full script search workflow
```

## Which tool to use for what

| I need to find...                          | Tool to call               | What it searches                |
|--------------------------------------------|----------------------------|---------------------------------|
| Scripts related to a feature by name       | `search_index()`           | `scripts_index.json` metadata   |
| A specific XPath or CSS selector           | `grep_scripts()`           | All `.py` files in `scripts/`   |
| Scripts semantically similar to a query    | `find_similar_scripts()`   | TF-IDF over script descriptions |
| The full code of a specific script         | `read_script()`            | A single `.py` file             |
| A list of all indexed scripts              | `list_scripts()`           | `scripts_index.json`            |
| An overview of a script's structure        | `get_script_outline()`     | Class/method names in a file    |

## How scripts are consumed

- The agent **never loads all 400 scripts at once** — it uses index + grep for targeted access.
- `search_index(query)` returns matching scripts from the pre-built JSON index (fastest).
- `grep_scripts(pattern)` runs regex across all script files (precise but slower).
- `find_similar_scripts(query)` uses TF-IDF scoring for semantic matching (best for natural language).
- `read_script(path)` reads a single file in full — only call this for top candidates.
